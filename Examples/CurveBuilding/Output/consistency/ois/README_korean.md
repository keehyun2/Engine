# OIS 일관성 검증 예제 설명 (Consistency OIS)

## 예제 개요

이 예제는 OIS(Overnight Index Swap) 곡선 구축의 **정확성을 검증**합니다.

### 일관성(Consistency)이란?

**정의**: 구축된 곡선이 입력 시장 데이터를 정확히 재현하는지 확인

**검증 방법**:
1. 시장 데이터로 OIS 곡선 구축 (부트스트래핑)
2. 동일한 곡선으로 OIS 상품 재가격
3. **NPV = 0**이면 일관성 있음 (정확히 재현)

### NPV가 0인 이유

```
구축된 곡선 → OIS 재가격
     ↓
NPV = 고정금리 PV - 변동금리 PV
     ↓
시장 데이터에서 금리 추출했으므로
NPV = 0 (at-par, 액면가 거래)
```

## 핵심 개념

### 1. 부트스트래핑(Bootstrapping) 검증

**과정**:
```
1. 시장 데이터 수집
   └─ OIS 금리 (1M, 3M, 6M, 1Y, 2Y, ...)

2. 곡선 구축
   └─ 단기 → 장기 순으로 할인율 계산

3. 재가격 테스트
   └─ 구축된 곡선으로 OIS 재가격

4. 일관성 확인
   └─ NPV ≈ 0 이면 성공
```

### 2. 왜 NPV가 0이어야 하는가?

**이론적 근거**:
```
시장 OIS 금리 = f(할인율 곡선)
              ↓
   같은 곡선으로 재가격하면
              ↓
   동일한 금리 도출
              ↓
   NPV = 0 (at-par)
```

**만약 NPV ≠ 0 이면?**
- 곡선 구축 오류
- 보간 방법 문제
- 컨벤션 불일치

### 3. 다양한 통화 검증

| 통화 | OIS 인덱스 | 특징 |
|------|-----------|------|
| EUR | EONIA | 유로존 익일 평균 |
| USD | FedFunds | 연방기금 금리 |
| GBP | SONIA | 파운드 익일 평균 |
| CHF | SARON | 스위스 프랑 익일 평균 |

## 실무 활용

### 1. 곡선 검증 절차

```python
def verify_curve_consistency():
    # 1. 곡선 구축
    curve = bootstrap_curve(market_data)

    # 2. 재가격
    npv = reprice_swap(curve, original_swap)

    # 3. 검증
    if abs(npv) < tolerance:
        return "PASS"
    else:
        return "FAIL"
```

## 📚 금융 용어 사전 (비전문가용)

### 기본 개념

**할인율 (Discount Factor/Rate)**
- 미래에 받을 돈을 현재 가치로 변환할 때 사용하는 비율
- 예: 1년 후 100원을 받는다면, 현재 가치는 100원 × 할인율 = 99원 (할인율 0.99인 경우)
- 금리가 높을수록 할인율은 낮아짐 (미래 가치 하락)

**이자율 곡선 (Yield Curve)**
- X축: 만기(기간), Y축: 금리로 그린 그래프
- 예: 1년물 금리 2%, 5년물 금리 3%, 10년물 금리 4%
- 시장의 미래 금리 전망을 반영

**부트스트래핑 (Bootstrapping)**
- 조각난 시장 데이터에서 전체 곡선을 만드는 기법
- 단기 금리 → 중기 금리 → 장기 금리 순으로 계산
- 마치 신발 끈을 묶듯 단계적으로 곡선 구축

**NPV (Net Present Value, 순현재가치)**
- 모든 미래 현금흐름을 현재 가치로 합산한 값
- 양수: 돈을 벌 거래, 음수: 돈을 잃을 거래
- 0: 공정 가격 (시장 가격과 정확히 일치)

### 상품 유형

**OIS (Overnight Index Swap, 익일 인덱스 스왑)**
- 고정 금리와 익일 금리(overnight rate)를 교환하는 파생상품
- 익일 금리: 중앙은행이 결정하는 단기 금리 (예: 한국의 콜금리)
- 기관들이 단기 금리 변동 리스크를 헷지할 때 사용

**FRA (Forward Rate Agreement, 선도 금리 계약)**
- 미래 특정 시점에 적용될 금리를 미리 정하는 계약
- 예: "3개월 후, 6개월간 금리를 3%로 고정"
- 금리 상승이 예상될 때 대출자가 사용

**FX 포워드 (Foreign Exchange Forward, 외환 선도 계약)**
- 미래 특정일에 고정된 환율로 통화를 교환하는 계약
- 예: "3개월 후 1달러당 1,300원에 교환"
- 환율 변동 리스크 회피에 사용

**베이시스 스왑 (Basis Swap)**
- 두 개의 변동 금리를 교환하는 스왑
- 예: "3M LIBOR 수취 / 6M LIBOR 지급"
- 서로 다른 금리 기준 간의 스프레드 교환

**크로스 통화 베이시스 스왑 (Cross-Currency Basis Swap)**
- 서로 다른 통화의 변동 금리를 교환 + 원금 교환
- 예: "USD 3M LIBOR 지급 / EUR 3M EURIBOR 수취"
- 통화 간 자금 조달 시 사용

### 시장 용어

**LIBOR (London Interbank Offered Rate)**
- 런던 은행 간 대출 금리 (금융계의 기준 금리)
- 2023년부터 SOFR 등으로 대체되었으나, 여전히 널리 사용

**EURIBOR (Euro Interbank Offered Rate)**
- 유로존 은행 간 대출 금리
- EURIBOR 3M: 3개물 유로 금리

**EONIA (Euro Overnight Index Average)**
- 유로존 익일 금리 평균
- 2021년부터 €STR로 대체

**FedFunds (연방기금 금리)**
- 미국 연방준비제도(Fed)의 기준 금리
- 미국 금리 정책의 핵심 지표

**SONIA (Sterling Overnight Index Average)**
- 영국 파운드화 익일 금리
- 영국의 기준 단기 금리

**SARON (Swiss Average Rate Overnight)**
- 스위스 프랑화 익일 금리
- 스위스의 기준 단기 금리

### 기술 용어

**베이시스 (Basis, 베이시스 포인트)**
- 두 금리나 가격 간의 차이
- 1bp = 0.01% (베이시스 포인트)
- 예: "EUR/USD 베이시스가 -20bp" → EUR가 USD보다 0.2% 불리

**커버드 이자율 평가 (Covered Interest Rate Parity)**
- 선도 환율 결정의 이론적 공식
- 현물 환율 × (1 + 외국 금리) / (1 +本国 금리) = 선도 환율
- 두 통화의 금리 차이가 선도 환율에 반영됨

**만기 (Maturity)**
- 파생상품 계약이 종료되는 날짜
- 예: "3M 만기" = 3개월 후 계약 종료

**명목 금액 (Notional Amount)**
- 계약의 기준이 되는 금액 (실제 교환되지 않음)
- 예: "명목 금액 100만 원" → 이자만 계산, 원금 교환 없을 수 있음

**DayCount (일수 계산 방식)**
- 이자 계산에 사용하는 기간 계산법
- Act/365: 실제 경과 일수 / 365
- 30/360: 한 달을 30일, 1년을 360일로 계산

**ATM (At-The-Money)**
- 옵션의 행사 가격이 현재 시장 가격과 동일한 상태
- "ATM 스왑" = 시장 금리와 동일한 고정금리를 가진 스왑

**재가격 (Repricing)**
- 구축된 곡선으로 다시 가격을 계산하는 과정
- 일관성 검증에 사용: "NPV = 0이면 곡선이 정확함"

### 통화 관련

**스왑 (Swap)**
- 두 당사자가 현금흐름을 교환하는 계약
- 금리 스왑: 고정금리 ↔ 변동금리
- 통화 스왑: 서로 다른 통화의 현금흐름 교환

**헷지 (Hedge)**
- 리스크를 줄이기 위한 대비책
- 예: "환율 상승을 예상 → USD 수익에 FX 포워드로 헷지"

**유동성 프리미엄 (Liquidity Premium)**
- 쉽게 현금화할 수 있는 자산이 더 높게 평가되는 현상
- 예: USD는 유동성이 풍부 → EUR/USD 베이시스가 USD에 유리하게 형성
