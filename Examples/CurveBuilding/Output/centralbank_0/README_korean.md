# 중앙은행 예제 출력 파일 설명 (Central Bank Example)

## 예제 개요

이 예제 시리즈는 **영국 중앙은행(BoE)의 SONIA 금리**를 사용하여 이자율 곡선을 구축하는 방법을 보여줍니다.

### SONIA란?

**SONIA = Sterling Overnight Index Average**
- 영국의 익일(Overnight) 인덱스 평균 금리
- 2021년부터 LIBOR를 완전히 대체
- 담보(Secured) 거래 기반
- OIS(Overnight Index Swap)의 기준

### CurveConfig #0~4의 차이

각 설정은 **Priority(우선순위)** 시스템을 다르게 구성:

| 설정 | Priority 0 | Priority 1 | Priority 2 | 특징 |
|------|-----------|-----------|-----------|------|
| #0 | 당일자 | BOE 회의일 OIS | 일반 OIS | **기본 설정** |
| #1 | 당일자 | BOE 회의일 OIS | - | 회의일만 사용 |
| #2 | 당일자 | - | 일반 OIS | 회의일 제외 |
| #3 | 당일자 | 장기 OIS | 단기 OIS | 장기 우선 |
| #4 | 모든 데이터 동일 Priority | | | 단일 우선순위 |

## 곡선 구축 원리

### Priority 기반 부트스트래핑

```
1. Priority 0: 당일자 예금으로 시작
   └─ MM/RATE/GBP/0D/1D (SONIA ON Rate)

2. Priority 1: BOE 회의일 OIS로 확장
   └─ IR_SWAP/RATE/GBP/20251218/1D/20260205
   └─ IR_SWAP/RATE/GBP/20260205/1D/20260319
   └─ ... (월별 회의일)

3. Priority 2: 일반 OIS로 완성
   └─ 3M, 4M, 5M, ..., 5Y (정기적 간격)
```

### BOE 회의일 OIS란?

**형식**: `IR_SWAP/RATE/GBP/{Spot}/1D/{회의일}`
- **Spot**: 계약 체결일 (예: 2025년 12월 18일)
- **회의일**: 금리 결정일 (예: 2026년 2월 5일)
- **1D**: 1일(Overnight) 인덱스

**예시**:
```
IR_SWAP/RATE/GBP/20251218/1D/20260205
```
- 2025년 12월 18일 체결
- 2026년 2월 5일 BoE 회의 결과 기준
- SONIA 평균 교환

### MinDistance 설정

- **목적**: 인접 데이터 간 최소 거리 보장
- **값**: 5일 = 데이터 간 최소 5일 간격
- **효과**: 곡선의 부드러움 유지

```python
if 만기 < 3년:

    interpolation = "Linear"
```

## 출력 파일 설명

### 1. npv.csv (순현재가치)

```
#TradeId,TradeType,Maturity,NPV
OIS,Swap,2032-09-22,#N/A
```

**NPV가 #N/A인 이유**:
- 예제 시점(2025-11-27) 기준으로 이미 진행 중인 계약
- 곡선 구축에 중점, 정확한 NPV 계산은 부가적

### 2. curves.csv (이자율 곡선)

| Tenor | Date | GBP | GBP-SONIA | 설명 |
|-------|------|-----|-----------|------|
| 1D | 2025-11-28 | 0.99989 | 0.99989 | 1일 만기 |
| 1W | 2025-12-04 | 0.99927 | 0.99927 | 1주 만기 |
| 1M | 2025-12-27 | 0.99612 | 0.99612 | 1개월 만기 |
| 1Y | 2026-11-27 | 0.96000 | 0.96000 | 1년 만기 |

**해석**:
- 할인율이 1에 가까울수록 단기 금리 낮음
- 1년 할인율 약 0.96 → 연간 약 4% 금리

## 핵심 개념

### 1. SONIA vs LIBOR

| 항목 | LIBOR (구) | SONIA (신) |
|------|-----------|-------------|
| 성격 | 은행 간 거래 금리 | 익일 인덱스 평균 |
| 리스크 | 은행 신용 리스크 포함 | 담보 기반 (무위험) |
| 확정 | 매영업일 | 익일 평균 |
| 적용 | 대출, 스왑 등 | OIS, 파생상품 |

### 2. OIS의 구조

```
OIS (Overnight Index Swap)
├─ 고정금리 다리: 연 4.204% 수신
└─ 변동금리 다리: SONIA 지급 (반기별 평균)
```

**금리 계산**:
```
SONIA 평균 = Σ(매일 SONIA 금리) / 영업일 수
복리 = (1 + SONIA1/365) × (1 + SONIA2/365) × ...
```

### 3. Priority 시스템의 중요성

**왜 Priority가 필요한가?**
- 시장 데이터의 신뢰성 차이
- BOE 회의일 OIS가 가장 신뢰성 높음
- 단기 OIS는 유동성 낮을 수 있음

**구축 순서**:
```
1. Priority 0 (항상 사용) → 당일자 확정
2. Priority 1 (가장 중요)  → 핵심 곡선 구축
3. Priority 2 (선택적)    → 곡선 보완
```

## 실무 팁

### 1. 중앙은행 정책 영향

- **BoE 회의일**: 매월 목요일 (예외 있음)
- **기준금리 발표**: 14:00 (런던 시간)
- **시장 반영**: 즉시 반영됨

### 2. SONIA 특징

- **무위험 금리**: 영국 국채 금리 근접
- **음수 가능성**: 매우 낮지만 이론적 가능
- **일일 확정**: 전날 23:59 확정

| 프로젝션 | GBP1D | SONIA 기반 |
| 리스크 프리싱 | GBP1D | 일관성 유지 |
